FAQ
Where am I?
This is a report testing assistance service. It's intended to help testers to run multiple isolated report instances.
How do I launch new report instances?
Enter the name of a Startrek ticket and click the Launch button (you'll be asked to introduce yourself the first time).
The service will check out the ticket, find the last comment of the form Report binaries stored at: full:path/to/the/report/binary, download the binary, find free ports, generate all necessary configuration files, and launch a market-parallel-report servant.
What does the status column say?
Active means that the servant's alive and is listening on the specified port.
Inactive means that although the servant's alive it is not listening on the port.
Dead means that the servant's dead and will be removed from the table once you click the Purge button.
How do I terminate an instance?
Select instances to be terminated and click the Terminate button (or the Kill button, as a last resort).
What is the Purge button for?
The Purge button removes dead instances from the table along with their binaries and logs.
Be careful with it.

How do I install the service?
report-testing-assistant$ sudo make install
How do I configure the service?
/etc/yandex/report-testing-assistant.cfg
The configuration file consists of option-value pairs separated by =. It can also contain comment lines that start with #.
See the rta.settings module for the list of available options.
How do I control the service?
$ sudo service report-testing-assistant (start|stop|restart)
Where is the service log?
/var/log/report-testing-assistant.log
What if I think this service sucks?
report-testing-assistant$ sudo make uninstall
